## 关键问题
1. 真实抓取是否默认启用？决策：默认关闭，避免仿真误触真实机械臂。
2. 真实抓取失败是否让仿真失败？决策：`real_arm.fail_on_error` 控制，默认 `1`。

## 已做决策
| 决策 | 理由 |
|------|------|
| 默认关闭真实抓取 | 接实物有安全风险，必须显式配置或环境变量启用 |
| 真实桥接接入 `slaver/robot/module/grasp.py` | 用户明确 LLM 工具入口调用这个文件；真实信号应在该入口层发送 |
| 真实抓取在仿真成功后同步执行 | 用户要求仿真也需要进行；同步等待可把真实结果返回给调用方 |
| 真实失败默认让 `/grasp` 返回失败 | 迁移到实物后调用方必须能看到物理执行失败；需要只看仿真时可设 `fail_on_error: 0` |
